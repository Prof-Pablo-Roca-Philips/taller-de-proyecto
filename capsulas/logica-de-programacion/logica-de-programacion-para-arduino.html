1 — Variables

Una caja con nombre

Una variable guarda un valor que puede cambiar. En Arduino, casi siempre ese valor viene de un sensor.

Ejemplo real

Un sensor de temperatura mide 24°C. Guardás ese número en una variable llamada temperatura para poder usarlo después en tu código.

int temperatura = 24;
💡 Por qué importa: sin variables, no podrías comparar el valor del sensor con un umbral, ni decidir nada. Toda la lógica del Arduino empieza guardando lo que el sensor midió.
Del sensor a la variable 0 / 6
Tipos de dato

¿Qué tipo de variable necesito?

El tipo depende de qué mide el sensor. Elegir mal el tipo es un error común de principiante.

int
Número entero
Distancia en cm, cantidad de vueltas. Ej: int distancia = 15;
float
Número decimal
Temperatura, humedad. Ej: float humedad = 63.5;
bool
Verdadero o falso
Presencia detectada o no. Ej: bool hayPersona = true;
Str
Texto (String)
Mensajes o estados. Ej: String estado = "OK";
💡 Regla práctica: un sensor de presencia (PIR) siempre es bool. Un sensor de temperatura o humedad casi siempre es float. Un sensor de distancia ultrasónico suele ser int.
Decisión: motor de la faja 0 / 7
2 — Condicionales

Si se cumple algo, hacé una acción

Un condicional (if) compara un valor y decide qué hacer. Es la herramienta que convierte una simple lectura de sensor en una decisión automática.

Recordá esta unión: este es el mismo umbral (72°C, motor de la faja) que vimos en Unidad 3. Ahí lo enunciamos como problema técnico — acá lo convertimos en código real.
Practicá Completá el condicional para un sensor de humedad: si la humedad del suelo baja de 40%, activar el riego.
Operadores

Los 6 operadores de comparación

Los usás siempre dentro de un if, comparando una variable con un valor.

>
Mayor que
Ej: temperatura > 72 — se activa cuando supera el umbral.
<
Menor que
Ej: humedad < 40 — se activa cuando cae por debajo.
==
Igual a
Ej: estado == true — compara si son exactamente iguales.
!=
Distinto de
Ej: sensor != 0 — se activa cuando hay cualquier lectura.
>=
Mayor o igual
Ej: nivel >= 100 — incluye el valor límite.
<=
Menor o igual
Ej: bateria <= 20 — incluye el valor límite.
⚠ Ojo con el = vs ==: un solo = asigna un valor ("guardá esto acá"), dos == comparan ("¿son iguales?"). Confundirlos es el error #1 de los que arrancan con Arduino.
3 — Loops

Repetir sin que se lo pidas de nuevo

Arduino ya repite loop() solo, para siempre, mientras esté encendido. Adentro de esa función podés agregar tus propias repeticiones con for o while.

loop() — automático

Se ejecuta una y otra vez sin parar. Ahí va la lectura constante de tus sensores.

for — contado

Repite una cantidad exacta de veces. Ej: tomar 10 lecturas y promediarlas.

💡 Ejemplo real: leer un sensor de distancia 5 veces seguidas y promediar el resultado reduce el ruido de la medición — mucho más confiable que una sola lectura.
Loop leyendo un sensor 0 / 6
Esqueleto de todo programa Arduino 0 / 8
4 — Funciones básicas

setup(), loop(), y las tuyas propias

Todo programa de Arduino necesita setup() (corre 1 vez) y loop() (corre para siempre). Además, podés crear tus propias funciones para ordenar el código.

  • setup() — configurás pines, inicializás sensores. Una sola vez.
  • loop() — la lógica principal, se repite sin parar.
  • Función propia — como leerTemperatura(), agrupa código que usás varias veces.
Por qué separar en funciones: si necesitás leer la temperatura en 3 lugares distintos del código, escribís la función una vez y la llamás las 3 veces. Menos errores, más orden.
5 — Entrada

Leer un sensor

Hay dos formas de leer, según el tipo de sensor.

digitalRead()

Lee solo 0 o 1 (sí/no). Ej: sensor de presencia PIR.

digitalRead(2);
analogRead()

Lee un rango de 0 a 1023. Ej: fotoresistor (LDR), sensor de humedad.

analogRead(A0);
💡 Ejemplo real: un LDR (fotoresistor) conectado al pin A0 devuelve un número más alto cuanta más luz recibe. Eso te permite decidir si encender una luz automática.
Sensor → Arduino → variable 0 / 6
Variable → Arduino → actuador 0 / 6
6 — Salida

Activar un actuador

Igual que con la entrada, hay dos formas de escribir según lo que necesites.

digitalWrite()

Prende o apaga (todo o nada). Ej: relay, LED, buzzer.

digitalWrite(8, HIGH);
analogWrite()

Controla intensidad (PWM, 0-255). Ej: velocidad de un motor, brillo de un LED.

analogWrite(9, 180);
💡 Ejemplo real: un servomotor que abre una válvula de riego usa digitalWrite para encenderlo, o analogWrite si necesitás controlar la velocidad de un motor DC.
