Trabajo en equipo

Scrum, Sprints y roles de equipo

La metodología ágil que organiza el trabajo en ciclos cortos, con entregas frecuentes y ajustes constantes.

Un Sprint es un ciclo corto de trabajo (1-2 semanas) con un objetivo concreto y entregable al final. Se planifica, se ejecuta, se revisa y se ajusta.

1. Planificar

Definir qué se va a lograr en este ciclo.

2. Ejecutar

Hacer el trabajo del equipo.

3. Revisar

Ver qué se logró y qué no.

4. Ajustar

Corregir el rumbo para el próximo ciclo.

Roles sugeridos en un equipo de 4-5

Líder de diseño

Planos, boceto, modelado CAD.

Líder de electrónica

Sensores, Arduino, cableado.

Líder de construcción

Herramientas, ensamblaje.

Líder de documentación

Registro, carpeta, presentación.

💡 Tablero simple: tres columnas — Por hacer / Haciendo / Terminado. Cada tarea es una tarjeta que se mueve de izquierda a derecha.
Concepto clave

¿Qué es un MVP?

Producto Mínimo Viable — lo mínimo que tiene que funcionar para demostrar que la solución es posible.

MVP no es la versión incompleta del producto. Es la versión más simple que ya resuelve el problema.
✗ No es un MVP

Un prototipo con todas las funciones imaginables, cada sensor posible, la carcasa perfecta y el diseño final terminado. Eso no es mínimo — es el producto final.

✓ Sí es un MVP

Un sensor + un actuador + la lógica mínima que demuestra que el objetivo SMART se cumple. Ejemplo: riego automático = sensor de humedad + bomba + Arduino.

⚠️ Definí tu MVP antes de bocetar. Eso acota todo lo que sigue — sketch, inventario y presupuesto se hacen sobre el MVP, no sobre la versión soñada.

Puente necesario

Lógica de programación — lo mínimo

Tinkercad ya simula código para tu circuito. Antes de seguir, un repaso mínimo de los conceptos que vas a necesitar.

Variables

Guardan un valor que cambia.

int temp = 0;

Condicionales

Si se cumple algo, hacé una acción.

if (temp > 72) {'{'}...{'}'}

Loops

Repetir una acción todo el tiempo.

while (true) {'{'}...{'}'}

Entrada / Salida

Leer un sensor o activar un actuador.

digitalRead / Write()
📚 Si necesitás más profundidad: existe una cápsula completa de Lógica de Programación para Arduino, con ejercicios interactivos. Consultá a tu docente si tu equipo la necesita.

Bloque C · Factibilidad

Presupuesto y factibilidad

El inventario dice qué necesitás. El presupuesto dice si es viable conseguirlo a tiempo y al costo correcto.

1. Mercado

Opciones y precios.

2. Proveedores

Dónde comprar cada pieza.

3. Cuellos de botella

Qué tarda más en llegar.

4. Costo-beneficio

¿Vale pagar más por llegar antes?

5. Decisión final

Costo vs tiempo de espera.

⚠️ Ejemplo real: un sensor específico puede ser el más barato pero tardar 3 semanas en llegar. Si tu proyecto tiene 2 semanas de plazo, el sensor "más caro pero disponible ya" es la decisión correcta.
