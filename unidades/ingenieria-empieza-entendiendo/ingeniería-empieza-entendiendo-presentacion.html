Concepto central

El sistema interactivo

Desde una puerta automática hasta un robot industrial — todo sistema automático funciona con el mismo principio.

Entorno → Sensor → Decisión → Acción0 / 4
Puerta automática

Presencia (infrarrojo) → detecta persona → abre la puerta automáticamente.

Aire acondicionado

Temperatura > 26°C → compara con umbral → activa el compresor.

Sensor de humo

Partículas en el aire → supera el nivel → dispara la alarma sonora.

💡 Tres componentes físicos: sensor (mide), controlador (decide), actuador (actúa). Todo sistema automático tiene estos tres.
Identidad profesional

¿Qué es un técnico electromecánico?

El técnico no solo opera máquinas — las diseña, las mejora y resuelve los problemas que generan.

Un técnico electromecánico es un profesional que combina conocimientos de electricidad, mecánica, electrónica y automatización para diseñar, construir y mantener sistemas que resuelven problemas reales. En la era de la Industria 4.0, también trabaja con robótica, sensores, programación e inteligencia artificial.
Industria 4.0

Fábricas inteligentes donde máquinas, sensores y sistemas se comunican entre sí. El técnico diseña y mantiene esos sistemas.

IoT

Internet de las Cosas. Dispositivos conectados que recopilan y comparten datos, desde sensores hasta robots colaborativos.

Automatización

Sistemas que funcionan solos, tomando decisiones basadas en datos del entorno. El núcleo de la electromecánica moderna.

IA aplicada

Algoritmos que permiten a los sistemas aprender y adaptarse. Clasificación, detección de fallas, optimización.

Punto de partida

El problema vago

Así describe un problema alguien que lo vive, pero no sabe cómo resolverlo técnicamente.

Un problema vago no tiene contexto, no tiene variable medible, no tiene umbral. Es una sensación o descripción informal. No se puede programar, no se puede medir, no se puede resolver sin antes transformarlo.
🔊 "Hay mucho ruido."

¿Dónde? ¿Cuánto ruido? ¿A partir de qué nivel es un problema? No se puede resolver así.

💡 "La luz molesta."

¿Qué luz? ¿Molesta cuánto? ¿A quién? ¿En qué condición exacta?

🚪 "La puerta queda abierta."

¿Qué puerta? ¿Cuánto abierta? ¿Por qué es un problema? ¿Qué consecuencia tiene?

⚠️ ¿Por qué no alcanza? Un problema vago no se puede medir, no se puede programar en un controlador y no se puede verificar si fue resuelto. Para diseñar una solución técnica, primero hay que convertirlo en un problema técnico.
💡 Test rápido: ¿Podés programar un Arduino para resolver "hay mucho ruido"? No — porque no sabés qué medir ni cuándo actuar. Eso te dice que todavía es un problema vago.
Herramienta central

Las 4 componentes del problema técnico

Todo problema técnico tiene exactamente 4 componentes. Ni más, ni menos. Identificarlas es lo que transforma el problema vago en algo resoluble.

1 · CONTEXTO
"¿En qué sistema, lugar o momento ocurre el problema?"
⚠ Trampa: confundir el contexto con el problema. El contexto es el escenario, no el síntoma.
Debe poder delimitarse o dibujarse. Si sigue siendo "en la sociedad", es demasiado amplio.
2 · VARIABLE
"¿Qué es lo único que, si lo mido, me dice si el problema existe?"
⚠ Trampa: elegir algo que no se puede medir directamente. La variable tiene que tener un sensor.
Tiene que ser observable o medible directamente. No puede ser una sensación.
3 · PUNTO DE QUIEBRE
"¿A partir de qué valor o condición digo que el problema empieza?"
⚠ Trampa: dejarlo subjetivo. "Cuando ya es mucho" no es un quiebre — es otra vagedad.
Un número, un estado lógico (sí/no), o un cambio concreto. Nunca subjetivo.
4 · PROBLEMA
"¿Qué efecto no deseado ocurre cuando se cruza ese umbral?"
⚠ Esta componente no genera información nueva: reconstruye el problema a partir de las tres anteriores.
Una sola oración, sin "y", sin adjetivos subjetivos como "mucho" o "grave".
💡 Orden importante: siempre identificar primero el CONTEXTO, luego la VARIABLE, luego el PUNTO DE QUIEBRE. El PROBLEMA sale solo al final como consecuencia lógica de los tres anteriores.
La herramienta

La fórmula del problema técnico

Una vez identificadas las 4 componentes, el enunciado técnico se construye con esta estructura. Cada hueco corresponde a una componente ya definida.

Estructura
En [CONTEXTO] ocurre [PROBLEMA] cuando [VARIABLE] supera [PUNTO DE QUIEBRE], lo que genera [EFECTO NO DESEADO].
✓Una sola oración — no usar "y" para unir dos problemas distintos.
✓Sin adjetivos subjetivos: no "mucho", "grave", "excesivo", "peligroso".
✓Cada término entre corchetes tiene que estar ya definido antes de escribir esta oración.
✗NO: "Hay mucho ruido en la calle que molesta a la gente y genera problemas."
✓SÍ: "El nivel de ruido en Av. 9 de Julio supera los 70 dB los viernes de noche, perturbando el descanso de los habitantes."
Para no repetir

Errores comunes

Los errores que aparecen siempre. Conocerlos de antemano te ahorra rehacer el trabajo.

🎯Contexto demasiado amplio
✗ Mal
"El contexto es la sociedad moderna con problemas de contaminación sonora."
✓ Bien
"Av. 9 de Julio, viernes y sábados de noche, entre las 22:00 y las 6:00 hs."
📏Variable que no se puede medir
✗ Mal
"La variable es la incomodidad de la gente con el ruido."
✓ Bien
"La variable es el nivel de ruido ambiental medido en decibeles (dB)."
⚖️Punto de quiebre subjetivo
✗ Mal
"Cuando ya hay demasiado ruido y la situación se pone grave."
✓ Bien
"Cuando el nivel de ruido supera los 70 dB — límite apto para la salud auditiva según OMS."
📝Dos problemas en un enunciado
✗ Mal
"El ruido molesta y también la luz encandila y eso hace que pasen accidentes."
✓ Bien
Una sola oración. Un solo problema. Si tenés varios, elegí el principal y enuncialo solo.

Caso resuelto

Ejemplo completo resuelto

El problema del ruido, resuelto paso a paso usando las 4 componentes y la fórmula.

PROBLEMA VAGO

"Hay mucho ruido."

CONTEXTO
Av. 9 de Julio, los viernes y sábados, entre las 22:00 y las 6:00 hs, en el entorno residencial adyacente.
VARIABLE
Nivel de ruido ambiental medido en decibeles (dB) — sensor: micrófono de precisión o sonómetro.
PTO. DE QUIEBRE
Nivel de ruido superior a 70 dB — límite máximo apto para la salud auditiva según la OMS.
PROB. TÉCNICO
El nivel de ruido en Av. 9 de Julio supera los 70 dB los viernes y sábados de noche, perturbando la atención y el descanso de los habitantes del entorno.
OBJETIVO
Detectar cuando el nivel de ruido supere los 70 dB y activar un sistema de alerta o registro para informar a las autoridades competentes.
DISPARADORA
¿Cómo podríamos detectar automáticamente cuándo el ruido en una avenida supera el límite seguro para la salud de los vecinos?

Respuestas de referencia

Cómo corregir el ejercicio

Modelo de respuesta y criterios de evaluación. No es la única respuesta válida — es una guía de corrección.

PROBLEMA A — RESPUESTA MODELO

"La luz molesta."

CONTEXTO
Autopista, carriles de circulación, durante la noche (20:00–6:00 hs).
VARIABLE
Ángulo y posición de los focos de los vehículos (en grados).
PTO. DE QUIEBRE
Ángulo de incidencia superior a 15° sobre el plano de visión del conductor.
ENUNCIADO
"La posición y ángulo de los focos de vehículos en la autopista supera los 15° de incidencia visual, generando encandilamiento y riesgo de accidente durante la conducción nocturna."
💡 Nota de evaluación: aceptar cualquier contexto coherente (autopista, aula con proyector, habitación). Lo importante es que el punto de quiebre tenga un valor concreto — no "cuando molesta mucho".

Criterios de evaluación — ambos problemas

✓El problema vago no tiene números ni valores técnicos.
✓El contexto tiene lugar, momento y actores identificados.
✓La variable tiene una unidad de medida (dB, °C, lux, cm, etc.).
✓El punto de quiebre es un número o condición objetiva — no "mucho" ni "excesivo".
✓El enunciado es una sola oración que usa la fórmula sin adjetivos subjetivos.
✗No aceptar puntos de quiebre subjetivos aunque el contexto sea bueno.
